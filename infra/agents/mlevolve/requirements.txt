# MLEvolve runtime requirements — curated minimal set.
#
# Build philosophy: follow MLE-Bench's environment/requirements.txt model.
# Hand-curate a small set of high-level packages we actually need; pin
# version-sensitive ones; let pip resolve transitives ONCE in this same
# base image at build time. No --no-deps gymnastics, no upstream
# kitchen-sink lockfile, no nvcc in the image, no compile-from-source
# packages.
#
# Source: enumerated by grep'ing every `^(import|from) X` line in
# /workspace/mlevolve/{agents,engine,llm,utils,config,run.py}. That gave
# us 22 third-party top-level imports. After filtering out modules our
# base image already provides via vllm/vllm-openai:v0.9.2 (torch, numpy,
# pandas, openai, requests, httpx, rich, jsonschema, omegaconf — all
# transitives of vllm or transformers), the new packages we need are
# below.
#
# Version pins:
# - google-genai: pinned for SDK stability. MLEvolve's upstream
#   requirements_base.txt had no pin; we use the same major as their
#   imports suggest (>=1.0).
# - faiss-cpu (NOT faiss-gpu): the global memory layer needs FAISS but
#   we disable it via use_global_memory: false in config.yaml. Include
#   anyway so that if a future cell enables it, it doesn't crash on
#   ImportError. -cpu variant ships prebuilt wheels with no CUDA
#   compile step (-gpu requires matching nvcc → exactly what bit us
#   with the kitchen-sink approach).
# - dataclasses-json: hyphenated pip name, underscored Python import
#   (`dataclasses_json`). MLEvolve's SearchNode serialization depends
#   on this for the journal.json schema.
#
# DELIBERATELY OMITTED (despite being in MLEvolve's import graph):
# - mlebench: only imported by engine/validation/format_server.py,
#   which we never load (use_grading_server: false). Verified absent
#   at build time via the assertion at the bottom of the Dockerfile.
#   (Originally also tried to omit `black` here; build-time smoke
#   caught that utils/response.py:4 imports it unconditionally — not
#   try/except-wrapped as I'd assumed. Now included below.)
#
# Regen recipe (if MLEvolve upstream adds new top-level imports):
#   docker run --rm -v $PWD:/work vllm/vllm-openai:v0.9.2 bash -c '
#     cd /work && \
#     grep -rhE "^(import|from) [a-zA-Z]" infra/agents/mlevolve/upstream/{agents,engine,llm,utils,config,run.py} \
#       | awk "{print \$2}" | awk -F. "{print \$1}" | sort -u'

# PEFT task family — needed by SAMSum / GSM8K / BoolQ tasks (added 2026-05).
# All pure-Python wheels (no nvcc, no compile-from-source); ~30 MB total.
# accelerate + datasets come transitively via transformers (vllm base ships
# transformers); peft + trl + bitsandbytes explicit because vllm doesn't
# depend on them (vllm has its own LoRA inference path that doesn't use
# peft for training). evaluate + rouge_score for summarization metrics.
#
# Versions PINNED to mid-2024 — matches MLE-Bench's environment (where
# MLEvolve was published) and the training-data prior of current LLMs.
# Loose pins (>=0.10) resolve to latest at build time and caused spike-009:
# trl 1.5.1 removed DataCollatorForCompletionOnlyLM, agent imported it from
# its prior, ImportError cascade. Pinning here closes that loop without
# advertising version specifics in the agent prompt.
peft==0.12.0
trl==0.9.6
bitsandbytes==0.43.3
evaluate
rouge_score
