INPUT: closed bug issues (known fixes), open issues (unresolved gotchas), and

  "Data & Tokenization", "Training Errors", "API / Version", "Other".
  Skip themes with zero applicable entries.

  **Fix:** <concrete action>. Include an issue link if the fix is non-obvious.

- Mark unresolved open issues with "**Known issue:**" instead of **Fix:**.
- Imperative fixes: "Upgrade X to Y", "Pass `--flag`", not "users should..."

- 15-40 bullets total. Quality over completeness.
